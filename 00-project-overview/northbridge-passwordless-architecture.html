Key Security Controls

  • Passwordless-first target state for users and administrators
  • Conditional Access evaluates user, device, location, risk, and application context
  • Identity Protection supports risk-based blocking and escalation
  • Emergency access accounts remain monitored and excluded from standard CA lockout paths
  • Phishing-resistant methods prioritized through Windows Hello for Business and FIDO2
  • Temporary Access Pass reserved for onboarding and recovery workflows
