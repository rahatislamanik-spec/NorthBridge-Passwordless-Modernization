Reviewer Explanation

This flow demonstrates a target Zero Trust sign-in path: device context, Conditional Access requirements, risk signals, and passwordless authentication are evaluated before Entra ID issues an access token. Conditional Access is shown as the policy engine, while Identity Protection supplies risk signals that can block, challenge, or route the user through remediation.

Success path: trusted device and context, CA requirements satisfied, acceptable risk, passwordless authentication succeeds, token issued, application access granted.

Block or remediation path: non-compliant device, failed policy, elevated risk, or failed authentication can block access, require stronger authentication, or route the user to recovery.
